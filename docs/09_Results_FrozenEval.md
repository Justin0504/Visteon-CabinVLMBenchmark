# 📊 核心实验结果 — 冻结测试集四方对比（论文消融）

> 测试集：`heldout_frozen.json`（每类 40 张、共 200，seed=123，**所有版本同集、永不进训练**）。
> 指标：车型/标志/景观 = 真值准确率(%)；OCR = 答案准确率(%)；车外场景 = LLM-judge(1–10)。

## 一、五方对比（base / v1 / v2 / v3 / v4，同一冻结集）

| 类别 | base | v1（车型+标志）| v2（+OCR景观无脑混）| v3（均衡）| v4（车型加权）|
|---|---|---|---|---|---|
| **车型 make/model** | 5.0 | **80.0** | 47.5 | 60.0 | 67.5 |
| 交通标志 | 92.5 | 100 | 100 | 100 | 100 |
| 自然景观 | 60.0 | 77.5 | **82.5** | **82.5** | 80.0 |
| 文字 OCR | 82.5 | 77.5 | **82.5** | **82.5** | 77.5 |
| 车外场景(judge) | 7.58 | **8.0** | 7.97 | 7.92 | 7.95 |

**当前最优模型**：
- **v3 = 最佳全才**（各类都不差、最低分最高、OCR+景观最强）→ **推荐交付**；
- **v1 = 车型专家**（80），但景观/OCR 覆盖弱；
- **v4**：车型加权使车型回升 60→67.5，但 OCR 掉回 77.5（此消彼长）。
- **核心发现**：多任务 LoRA 存在**能力互斥**——细粒度车型与 OCR 抢容量，数据配比是真实权衡旋钮，单模型难全赢。
- **下一步（进行中）**：两阶段（caption 知识注入 → QA finetune）= v5，尝试打破权衡（老师建议的方法）。

## 二、关键发现（论文级）
1. **标签引导微调极有效**：车型 base 5% → v1 **80%**（+75 分），标志 92.5%→100%。
2. **多任务数据混合对细粒度任务有干扰**：加入 OCR+景观后，车型从 v1 的 80% **退化到 v2 的 47.5%**（灾难性遗忘/稀释）。
3. **均衡采样能缓解但未根治**：v3（每源限 1200）把车型从 47.5% 回升到 60%，且景观/OCR 保持 82.5%——但仍低于 v1 的 80%。
4. **覆盖 vs 精度的权衡**：v1 车型/车外最强但缺景观/OCR；v2/v3 全类覆盖但车型受损。**v3 是当前最优"通才"**（各类都不差、覆盖最全）。

## 三、优化轨迹
```
base → v1(标签引导, 车型5→80) → v2(加数据但退化) → v3(均衡修复, 车型→60) → v4(车型加权, 目标恢复80+且保覆盖)
```

## 四、下一步（v4，进行中）
v3 车型 60 < v1 80 的差距 = 车型被稀释。**v4 对策：车型全量(加权)、其余下采样到 900**，让车型不被淹没，目标"既全覆盖、车型又回 80+"。跑完出 base/v1/v2/v3/v4 五方对比，定最终最优模型。

## 五、备注（评测严谨性）
- v1/v2 早期评测用过不同 held-out（不可跨比）；**本表 v1/v2/v3 均在统一冻结集上重评**，可信可比。
- 裁判为 Qwen2.5-VL 自评，存在偏置；车型/标志/景观用**客观真值准确率**已规避；车外/OCR 后续可上更强外部裁判。
